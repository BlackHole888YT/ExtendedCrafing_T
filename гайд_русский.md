# Руководство по созданию рецептов для Расширенного Верстака (1.21.1) от BlackHole888YT


По всем багам писать на:<br>
📞 Discord: blackhole888yt (Звонки только для русскоговорящих)

---

Этот гайд описывает полную структуру JSON-файлов для кастомного типа рецептов `extended_crafting_t:extended_shaped`. Рецепты поддерживают сложнейшие RPG-условия (время суток, измерения, ХП игрока, трату опыта, прочность инструментов и кастомные зачарования книг).

---

## 1. Общая структура рецепта (Корневой уровень)

Файлы рецептов должны располагаться по пути:  
📂 `[ИМЯ_ДАТАПАКА]/data/extended_crafting_t/recipe/[имя_рецепта].json`

### Пример полной структуры:
```json
{
  "type": "extended_crafting_t:extended_shaped",
  "category": "misc",
  "key": { ... },
  "additional_conditions": { ... },
  "pattern": [ ... ],
  "result": { ... }
}
```

### Корневые параметры:

| Параметр | Тип | Обязателен? | Описание |
| :--- | :--- | :---: | :--- |
| **`type`** | String | **Да** | Должен быть равен строго `"extended_crafting_t:extended_shaped"`. |
| **`category`** | String | **Да** | Ванильная категория рецепта (обычно `"misc"`, `"equipment"`, `"combat"`). |
| **`pattern`** | List of Strings | **Да** | Шаблон сетки крафта 3х3 (строки из 1-3 символов). |
| **`key`** | Object | **Да** | Словарь, привязывающий символы из `pattern` к свойствам ингредиентов. |
| **`additional_conditions`**| Object | Нет | Блок условий окружающей среды и характеристик игрока. Если опущен — крафт доступен всегда. |
| **`result`** | Object | **Да** | Описание получаемого предмета. |

---

## 2. Блок Ингредиентов (`key`)

Каждый символ из шаблона `pattern` настраивается индивидуально.

### Параметры внутри каждого символа (например, `"S"`):

| Параметр | Тип | Обязателен? | Дефолтное значение | Описание |
| :--- | :--- | :---: | :---: | :--- |
| **`ingredient`** | Object | **Да** | — | Стандартный ванильный ингредиент. Принимает `"item"` (конкретный предмет) или `"tag"` (тег материалов). |
| **`count`** | Integer | Нет | `1` | Минимальное количество предметов в слоте, необходимое для крафта, а также количество, списываемое при крафте. |
| **`consumable`** | Boolean | Нет | `true` | Если `false` — предмет останется лежать в сетке крафта после завершения (например, инструменты). |
| **`deal_damage`** | Integer | Нет | `0` | Сколько единиц прочности потеряет предмет при крафте. Число не должно превышать макисмальную прочность меча, иначе крафт не сработает. (работает только если `consumable` равен `false`). |
| **`work_enchantments`** | Boolean | Нет | `false` | Если `true`, то урон прочности (`deal_damage`) будет уменьшаться под действием ванильного зачарования Прочности (Unbreaking) на предмете. |
| **`book_id`** | String | Нет | — | Используется **только для зачарованных книг** в ингредиентах. Строго сверяет ID зачарования (например, `"minecraft:sharpness"`). |
| **`book_lvl`** | Integer | Нет | — | Используется совместно с `book_id`. Строго проверяет уровень чар на книге. Если уровень выше/ниже или на книге есть другие чары — рецепт заблокируется. |
| **`item_after_consume`** | Object | Нет | — | Предмет, который заменит ингредиент в слоте, если тот сломался (или полностью потратился). Полезно для возврата пустых ведер или «осколков» сломанных инструментов. |

#### Пример сложного ингредиента (меч, теряющий прочность и оставляющий книгу):
```json
"S": {
  "ingredient": { 
    "item": "minecraft:iron_sword" 
  },
  "consumable": false,
  "deal_damage": 150,
  "work_enchantments": true,
  "item_after_consume": { 
    "id": "minecraft:enchanted_book",
    "count": 1,
    "components": {
      "minecraft:custom_data": {
        "book_id": "minecraft:unbreaking",
        "book_lvl": "2"
      }
    }
  }
}
```

## 3. Блок Дополнительных Условий (`additional_conditions`)

Все параметры внутри этого блока являются **необязательными**. Если параметр опущен, проверка по нему не производится.

### Доступные условия:

| Параметр | Тип | Описание |
| :--- | :--- | :--- |
| **`required_dimension`** | String | Измерение, в котором должен физически стоять верстак (например, `"overworld"`, `"the_nether"`, `"the_end"`). |
| **`required_advancement`**| String | Полный ID достижения, которое должно быть получено у игрока (например, `"minecraft:story/mine_diamond"`). |
| **`required_hp`** | String | Требование к здоровью игрока. Поддерживает:<br>• `"max"` (строго полные ХП)<br>• `"min"` (меньше или равно 1 ХП / полсердца)<br>• `"half"` (ровно половина здоровья ±1.5 ед. допуска)<br>• Проценты (например, `"50%"` — не менее половины здоровья)<br>• Числа (например, `"15.0"` — не менее 15 единиц ХП). |
| **`xpCost`** | Object | Требование к опыту игрока в **поинтах** (точках опыта).<br>• `"value"` (Integer): сколько поинтов требуется.<br>• `"consumable"` (Boolean): списывать ли этот опыт при крафте. |
| **`xpLvl`** | Object | Требование к опыту игрока в **уровнях**.<br>• `"value"` (Integer): сколько уровней требуется.<br>• `"consumable"` (Boolean): списывать ли эти уровни при крафте. |
| **`required_time`** | Object | Требование к времени суток в игровом мире (в тиках).<br>• `"value"` (String): текстовый пресет (`"sunrise"`, `"day"`, `"noon"`, `"sunset"`, `"night"`, `"midnight"`) **ИЛИ** точное число в тиках (например, `"12500"`).<br>• `"interval"` (Integer): допуск в тиках в обе стороны (например, `500` означает `Целевое_Время ± 500 тиков`).<br>• `"interval_value"` (String): всегда `"tick"`. |

#### Пример блока условий:
```json
"additional_conditions": {
  "required_dimension": "overworld",
  "required_advancement": "minecraft:story/mine_diamond",
  "required_hp": "max",
  "xpCost": { "value": 150, "consumable": true },
  "xpLvl": { "value": 15, "consumable": false },
  "required_time": {
    "value": "12500",
    "interval": 500,
    "interval_value": "tick"
  }
}
```

## 4. Блок Результата (`result`)

Описывает предмет, получаемый на выходе. Полностью поддерживает стандарты Data Components версии 1.21.1.

### Параметры:

| Параметр | Тип | Обязателен? | Описание |
| :--- | :--- | :---: | :--- |
| **`id`** | String | **Да** | ID получаемого предмета (например, `"minecraft:enchanted_book"`). |
| **`count`** | Integer | Нет | Количество предметов на выходе (по умолчанию `1`). |
| **`components`** | Object | Нет | Блок компонентов данных 1.21.1. Используется для генерации кастомных зачарованных книг через `"minecraft:custom_data"` (с ключами `"book_id"` и `"book_lvl"`). |
|**`book_id`**| String | Нет | Айди книги, которую вы хотите получить. (советую погуглить чтоб понять какой id у какой книги)|
|**`book_lvl`** | String | Нет | Уровень зачарования книги, которую вы хотите получить. (советую не превышать максимальный уровень, доступный в обычной игре, иначе наковальная наложит вместо `Острота VII` `Острота V`)

#### Пример результата (Зачарованная книга на Починку I):
```json
"result": {
  "count": 1, -> тут можно даже 2 штуки получить
  "id": "minecraft:enchanted_book",
  "components": {
    "minecraft:custom_data": {
      "book_id": "minecraft:mending",
      "book_lvl": "1" -> тут тоже любой уровень
    }
  }
}
```
